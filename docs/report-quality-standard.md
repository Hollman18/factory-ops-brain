# Report quality standard

## Purpose
Define the minimum quality expected from generated reports.

## Every meaningful report should try to include
1. what happened
2. how important it is
3. what is driving it
4. what should happen next

## Quality criteria
- role-appropriate framing
- clear structure
- correct unit usage
- explicit comparison basis when relevant
- honesty about uncertainty or missing data
- recommendation or next-step guidance

## Rule
Reports should not become raw metric dumps unless the user explicitly asks for that style.
