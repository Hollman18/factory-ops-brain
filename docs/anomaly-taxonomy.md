# Anomaly taxonomy

## Purpose
Provide a shared language for describing abnormal behavior.

## Common anomaly types
- sharp drop
- sustained deterioration
- unexpected spike
- cross-entity outlier
- recurring instability
- target-risk pattern
- quality deviation pattern
- failure-risk pattern
- process drift

## Rule
The taxonomy should help the agent explain whether it is seeing noise, a one-off event, a sustained issue, or an emerging operational risk.
