# Global use-case matrix

## Purpose
Map the most relevant industrial use cases this template should be able to support across different manufacturing contexts.

## Universal use-case families

### 1. Performance monitoring
- current OEE
- current production pace
- current target progression
- current downtime / technical status

### 2. Historical comparison
- today vs yesterday
- week vs week
- month vs month
- shift vs shift
- plant vs plant
- line vs line
- machine vs machine
- reference vs reference

### 3. Exception detection
- KPI outside expected range
- deterioration vs baseline
- recurring abnormal pattern
- outlier entity or period

### 4. Root-cause direction
- what is most likely driving the change
- which entity explains the difference
- which KPI or loss pillar explains the issue

### 5. Predictive monitoring
- target-miss risk
- failure-risk escalation
- worsening quality/process stability
- expected continuation of deterioration if nothing changes

### 6. Role-based reporting
- executive report
- daily operational report
- shift-close report
- technical maintenance report
- quality deviation report

### 7. Entity drill-down
- executive summary -> plant -> area -> line -> machine -> shift -> reference
- reverse roll-up for high-level reporting

### 8. Missing-data handling
- source not connected
- partial source
- inconsistent source
- ambiguous entity request

## Cross-industry applicability
These use-case families are intended to apply across food, beverage, dairy, packaging, plastics, automotive, pharma, metals, textiles, continuous-process industries, and discrete manufacturing.

## Rule
The agent should remain useful even when a specific deployment emphasizes only a subset of these use cases at first.
