# Comparative analysis module

## Purpose
Support comparison across periods, shifts, entities, references, and hierarchy levels.

## Operational meaning
Comparative analysis helps the user understand not only what performance is, but how it changed and where the gap is most important.

## Expected analytical behavior
- compare period vs period
- compare entity vs entity
- compare shift vs shift
- compare reference vs reference
- compare plant vs plant
- rank best and worst performers
- identify the main driver behind the difference
- explain whether the difference is meaningful, a true deviation, or small/noisy

## Role-aware interpretation
- Manager/director: focus on strategic differences and impact
- Supervisor: focus on actionable gaps and deviations
- Operator: focus on simple operational interpretation when relevant

## Missing-data rule
If a requested comparison lacks valid data for one or more sides, say so clearly and do not fabricate conclusions.
