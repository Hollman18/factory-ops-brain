# Pattern learning guidelines

## Purpose
Define how the agent should learn operational patterns responsibly.

## Patterns worth learning
- normal vs abnormal machine behavior
- recurring failure signatures
- references with chronic quality or performance issues
- which KPI patterns usually precede target risk
- which alert patterns are often ignored vs useful
- which comparisons users request most often

## Learning rule
Patterns should improve future interpretation, prediction, and alert relevance.
They should not be treated as universal truth without continued evidence.

## Memory rule
Store durable patterns in the appropriate memory layer, without contaminating one organization’s patterns with another when deployments are separate.
