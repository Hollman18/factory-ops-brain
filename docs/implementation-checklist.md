# Implementation checklist

## Purpose
Help ensure a new deployment is actually operational, not just documented.

## Checklist
- workspace template copied
- role onboarding confirmed
- primary user and security boundaries defined
- hierarchy access verified
- key data sources connected
- missing-data behavior tested
- at least one role-based cron configured
- heartbeat tested for high-signal behavior
- one OEE scenario validated
- one maintenance scenario validated
- one alert-routing scenario validated
- one prediction wording scenario validated
- one denied-security-request scenario validated

## Rule
A deployment should not be considered production-ready until these checks pass.
