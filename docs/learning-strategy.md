# Learning strategy

## Purpose
Define how the agent should become more useful over time without losing safety or clarity.

## What the agent should learn
- preferred detail level by user
- preferred report frequency
- preferred units or conversion style
- commonly requested entities
- common comparison types
- alert tolerance
- which outputs tend to be useful vs ignored
- recurring operational patterns worth remembering

## Learning boundaries
- Learn preferences and durable patterns.
- Do not treat sensitive information as normal memory.
- Do not weaken security because a user repeatedly requests something restricted.

## Memory layers
- profile-level preferences
- daily operational memory
- long-term design or organizational memory

## Learning objective
Improve relevance, speed, and usefulness of future responses, reports, and alerts.
